roam

Leave a working agent on a remote box.

Dispatch an autonomous agent to a VM, let it work while you're away, and attach back any time to inspect and steer it. The agent replicates itself onto the box — it's one static binary, no Python, no runtime, nothing to install on the target. Built in machin (MFL).

GitHub → Download binary Changelog

Quickstart

# get a key for your provider (Anthropic, or any OpenAI-compatible endpoint)
export ROAM_API_KEY=sk-...

# send an agent to a box and walk away
roam send --to my-vm --allow-shell --confirm --verify \
  --goal "clone the repo, run its tests, write results.md"

# watch it think and act, live over ssh
roam attach my-vm

# it parks on anything destructive — approve out-of-band
roam approve my-vm

roam stop my-vm
🤖

A real agent, with tools

A genuine tool-use loop — read, write, run shell — until the goal is met. Claude or any OpenAI-compatible model (OpenRouter, …).

🛡️

Walk-away safe

Hard token & iteration budgets, a workdir sandbox, shell off by default, and a full journal of every step.

✋

Human-in-the-loop

The confirm-gate parks destructive commands for async approval; a deny budget stops a stubborn agent. Same interface for humans and supervising agents.

🔍

Verified completion

An optional judge checks the goal against real evidence — the working directory and action log — not the agent's word. Fail-open, so it never blocks correct work.
