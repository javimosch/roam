## [Unreleased]

### Added
- **`--hub` — approve from your phone.** Point a job at a
  [roam-panel](https://github.com/javimosch/roam-panel) hub (`--hub <url> --hub-token`, or
  the `ROAM_HUB_TOKEN` env var) and the worker mirrors its journal + status to the hub and
  polls it for approve/deny/steer/stop decisions — so a parked destructive command can be
  approved from a mobile web page (or a one-tap email link), no ssh. Fully opt-in (no `--hub`
  = nothing reported); the token rides as an env var, not an arg. Reporting works for all
  engines; per-command approve/deny is for the anthropic/openai engines (`debri` can't park).
  Implemented as `hub_sync`/`hub_poll`/`worker_close` reusing the existing `http_post_retry`.
- **`--provider debri` — a subscription engine (no per-token cost).** A third engine
  alongside `anthropic`/`openai`: instead of roam's own LLM tool-loop, the worker delegates
  the whole goal to the [`devin`](https://devin.ai) CLI (SWE-1.x) via
  [debri](https://github.com/javimosch/debri), tailing its JSONL stream into the journal —
  keeping roam's dispatch / self-replication / `attach` / `stop`. The agent runs on a flat
  Devin subscription, so long open-ended builds don't burn per-token budget; no API key
  required. Flags `--debri-perm auto|dangerous` (default `dangerous`) and `--debri-stable
  <ms>` (silence safety-cap, default `300000`); a done-marker is appended to the goal so a
  cooperative session finishes promptly. `stop` SIGTERMs debri so it tears down the devin
  session cleanly. Proven building [machin-hill-climb](https://github.com/javimosch/machin-hill-climb)
  (a Box2D+raylib game + a pure-MFL WebAssembly port) on SWE-1.7.

## [0.1.0] — 2026-07-10

First release. Leave a working agent on a remote box — one static binary, no Python on the target.

### Added
- **Remote dispatch + self-replication.** `roam send --to <host>` scp's the binary to the
  box and boots a detached worker there; control ops (`status`/`attach`/`steer`/`stop`) run
  over ssh against the remote's own SQLite journal. Verified end-to-end on a real remote.
- **LLM tool-loop.** The worker runs a real agentic tool-use loop (`read_file`, `write_file`,
  `finish`, and gated `run_shell`), written in machin itself — raw HTTP, no SDK.
- **Two providers.** `--provider anthropic` (Claude Messages API) and `--provider openai`
  (any OpenAI-compatible endpoint, e.g. OpenRouter). Default model `claude-opus-4-8`.
- **Trust layer.** Hard `--max-iters` / `--tokens` budgets, a per-job workdir sandbox
  (file tools confined; `..`/absolute refused), shell off unless `--allow-shell`, and a full
  append-only journal of every model turn, tool call, result, and token cost.
- **Confirm-gate (`--confirm`).** Destructive shell commands park for async approval
  (`roam approve` / `deny` / `stop`); a deny budget (`--max-denials`) auto-halts a stubborn
  agent. Live-proven on a remote box, including catching a `sudo rm` escalation.
- **Goal-verify (`--verify`).** A `finish` is checked by an independent judge that decides
  from evidence (real workdir + action log), not the agent's self-report; fail-open, with a
  verify budget (`--max-verify`).
- **Static release binary.** `make release` produces a fully-static `roam-x86_64-linux`
  (bundled SQLite + static OpenSSL + CA store) that runs FROM scratch on any x86-64 Linux.
